## 技能說明
- **核心描述**：精通運用 Docker 容器化技術隔離、封裝微服務環境，並透過 Docker Compose 進行多服務的一鍵編排。在此基礎上更結合 CI/CD 自動化部署流程，最大化提升交付能效、環境一致性與可靠度。
- **熟練程度**：精通 / 熟練
- **核心技術**：Docker, Docker Compose, GitLab CI/CD, 自動化部署, 微服務編排

## 實際專案紀錄

### 多微服務容器化與標準部署環境建置
- **情境 (Situation)**：專案中包含後端（FastAPI）、前端（Next.js）與向量資料庫（Qdrant）等多個服務，傳統手動部署方式繁瑣、環境組態複雜且容易出錯。
- **任務 (Task)**：實作環境標準化，推動容器化部署以降低部署成本。
- **行動 (Action)**：運用 Docker 容器化技術，為前端、後端服務分別撰寫 `Dockerfile`；進一步採用 Docker Compose 統一管理與定義多容器生態系，完成一鍵即可啟動的開發與部署配置。
- **結果 (Result)**：極大簡化了跨團隊開發和運維部署之流程，打造高一致性的環境標準，免去繁多繁瑣的手動操作。

### GitLab CI/CD 自動化部署流程編排
- **情境 (Situation)**：在多專案管理情境下，傳統手動編譯與配置發布程序較為繁瑣，容易發生人為組態疏漏（手動配置錯誤），影響交付效率與維運平穩性。
- **任務 (Task)**：設計並建置自動化的 CI/CD 部署流水線（Pipeline），大幅縮減手動介入與配置錯誤。
- **行動 (Action)**：編寫並編排 GitLab CI/CD 持續整合與部署流水線，自動執行程式碼檢查、打包、Docker 映像檔構建與遠端伺服器一鍵部署，將整個交付生命週期自動化管理。
- **結果 (Result)**：順利將人工配置錯誤率顯著降低達 30%，同時大幅提高系統交付的週轉效率與多次迭代部署的穩定度。
