## 技能說明
- **核心描述**：在跨職能協作團隊中扮演核心角色，負責整體系統架構、資料庫架構（Schema）和 API 規格之規劃，並擅長撰寫專業、清晰的技術系統分析（SA）文檔來促成開發工程之流暢進行。
- **熟練程度**：熟練
- **核心技術**：System Architecture, Database Schema Design, API Specification, SA Document (系統分析文件)

## 實際專案紀錄

### 跨團隊系統架構規劃與 SA 技術文檔主導
- **情境 (Situation)**：在 4 至 6 人的工程研發團隊協同下，多個新型或大型專案存在架構不明晰、資料庫架構未經標準規劃，以及 API 交換格式紊亂等情境，影響團隊開發效率與未來系統之可維護度。
- **任務 (Task)**：主導整體系統之架構、資料庫綱要（Database schemas）及 API 技術規格之設計，並建立可信的系統分析（SA）技術文件。
- **行動 (Action)**：細緻規劃軟體系統整體物理與邏輯架構，設計高性能關聯式資料表索引關係（PostgreSQL / SQL Server），負責 API 規格文檔之定義，並撰寫結構完整、容易讀懂的系統分析（SA）說明書，引導開發流程標準化。
- **結果 (Result)**：提供團隊極具參考價值的技術分析文檔，縮短溝通壁壘，減少多維度協作摩擦，顯著增強新進工程師對專案了解與架構長期維護性。
